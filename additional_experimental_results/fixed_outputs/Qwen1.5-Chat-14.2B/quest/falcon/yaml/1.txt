The issue is with the `tasks` key having a mapping value instead of an array. Here's the corrected YAML:

```yaml
game:
  description: quest description
  id: <i.e. id>
  objective: <i.e. quest objective>
  quest_giver: <i.e. quest giver>
  reward: <i. e. quest reward>
  tasks:
    - description: task description
      location: <i. e. task0 location, optional>
      objective: task objective
      order: task order
  title: <i. e. quest title>
```

I removed the `type.0` and fixed the indentation for the `tasks` array item. If you want to make `location` optional with a default value, you can add a condition like this:

```yaml
game:
  description: quest description
  id: <i.e. id>
  objective: <i. e. quest objective>
  quest_giver: <i. e. quest giver>
  reward: <i. e. quest reward>
  tasks:
    - description: task description
      location: &default_location "task location"
      objective: task objective
      order: task order
      location: *default_location  # Use the default if not provided
  title: <i. e. quest title>
```
