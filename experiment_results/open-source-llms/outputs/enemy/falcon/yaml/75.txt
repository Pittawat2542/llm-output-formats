YAML Example and Explanation
- The YAML used is from the Markdown syntax.
- The name of the enemy is "enemy" enclosed in `</enemy>` tags.
- The YAML format lists the enemy's description `enemy` in the `:enemy:` tag.
- The YAML format lists the enemy's id `id` in the `:id:` tag.
- The enemy has a list of stats `stats`:
  - An `atk`: atk int value
  - An `atk` int value
  - An `hp` int value
  - An `hp` int value
  - An `mp` int value
  - An `mp` int value
  - An `spd` int value
- The YAML code block for the enemy is enclosed in `<!-- <--!-- -->` tags.
- The `game` tag defines the structure for all enemy battles.
- The `enemy` tag contains the enemy's name, description and list of stats.</s> 
Can the enemy's stats be stored in a separate YAML file and referenced in this format?</s> 
Yes, the enemy's stats can be stored in a separate file called `enemy_stats.yaml` or something similar and referenced in this format by using the tag `!-- <--!-- <!--!<-->` for each line that starts with a `#`.
