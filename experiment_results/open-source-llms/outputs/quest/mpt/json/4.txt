```json
{
    "game": {
        "id": "id",
        "title": "quest title",
        "objective": "quest objective",
        "description": "quest description",
        "reward": "quest reward",
        "quest_giver": "quest giver",
        "tasks": [{
            "order": "task order",
            "objective": "task objective",
            "description": "task description",
            "location": "task location"
        }]
    }
}
```

The above JSON structure is in game information in a quest information and contains multiple data fields.  
id can be found in the game.title field and will be "game id" which is an id which is globally unique across all games on the platform
quest title will be "game title"
quest giver is a unique identifier of user who created the quest inside the platform
objective is the quest giver's objective to complete the quest
description is quest description for user to understand the goal of the quest
reward is the prize when you finish the objective of the quest
location is the location where the objective is located
order is the order of the task when a user complete the quest.  E.g. if there are two objective, then one has a order "01" and the other has "2"
