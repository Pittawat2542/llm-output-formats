The quest can be returned in XML format. All objects should be wrapped in their respective class names to ensure that the output format is maintained. The task names, objective, location, and rewards can be added in the task-level data. The quest type and items can be added as a parent node, and all the objectives and their corresponding data can be added as child nodes. The tasks can be represented as child nodes and the task givers as child nodes with unique class names. The tasks and their givers should be represented as child elements, and the tasks' rewards can be represented as a reward element.
